## What we are building

A UCP-compliant MCP server exposing Tesco's commerce capabilities to AI agents
(Gemini, OpenAI, Claude, LangGraph etc). Lives in a monorepo alongside the
`gql-to-mcp` SDK package.

**Current goal:** implement `search_catalog` and `lookup_catalog` as the first
two UCP tools, wired to Tesco's GraphQL federation via `gql-to-mcp` SDK.

## Architecture decisions

### Types split — two kinds, two locations

**UCP types → `src/types/ucp.ts` (common, shared)**
`UcpProduct`, `UcpVariant`, `UcpMoney`, `UcpMessage`, response shapes.
Defined by the UCP spec. Every tool returns these same shapes.
Check `@ucp-js/sdk` first — if it exports catalog types, import from there.

**GQL types → `src/graphql/generated/` (generated, per-capability)**
`GqlProductItem`, `GqlSearchResponse`, query variable types.
Auto-generated by graphql-codegen from your query files.
Never write these by hand.

### Queries live in `src/graphql/queries/`

One file per capability. These are the codegen source files.
Queries rarely change independently of the tool — but keeping them
separate means codegen can read them without importing tool logic.

### One `tool.ts` per capability

Each tool file contains: Zod inputSchema, cursor helpers, mapper
(GQL → UCP), execute(), exported CustomTool object.
Only split the mapper out when it grows past ~80 lines.

### No separate GraphQL client

`ctx.gql<T>()` from the SDK handles everything: HTTP, headers, JWT.
Just pass the query string and variables.

### Zod for all input schemas

SDK converts Zod → JSON Schema at registration. Never write plain
JSON Schema for tool inputs.

## SDK contracts

### `CustomTool`

```typescript
export interface CustomTool {
  name: string;
  description: string;
  authMode?: 'jwt' | 'anonymous';
  inputSchema: ToolInputSchema | ZodObject<ZodRawShape>;
  execute(args: Record<string, unknown>, ctx: ToolContext): Promise<unknown>;
}

export interface ToolContext {
  gql<T = unknown>(operation: string, variables?: Record<string, unknown>): Promise<GqlResult<T>>;
  headers: Record<string, string>;
  jwt: JwtPayload | undefined;
  traceId: string;
}
```

### SDK changes needed (if not already done)

**1. Zod support in `registerCustomTools`**

```typescript
// packages/gql-to-mcp/src/registry/tool-registry.ts
import { z } from 'zod';
import { zodToJsonSchema } from 'zod-to-json-schema';

// in registerCustomTools(), before storing:
inputSchema: tool.inputSchema instanceof z.ZodType
  ? zodToJsonSchema(tool.inputSchema as any) as ToolInputSchema
  : tool.inputSchema,
```

**2. `listTools()` on public server**

```typescript
// in types/index.ts — add to McpServer interface:
listTools(): ToolManifestEntry[];

// in index.ts — add to createMcpServer return:
listTools() { return toolRegistry.getAllTools(); },
```

## Task 1: `search_catalog`

**Spec:** https://ucp.dev/draft/specification/catalog/mcp/#search_catalog

### UCP request — what the agent sends

| Param                   | Required | Notes                          |
| ----------------------- | -------- | ------------------------------ |
| `query`                 | **YES**  | search term                    |
| `filters.in_stock_only` | no       | default true                   |
| `filters.brand`         | no       | single brand                   |
| `filters.brands`        | no       | multiple brands                |
| `filters.dietary`       | no       | e.g. "vegan"                   |
| `filters.dietaries`     | no       | multiple                       |
| `filters.offers`        | no       | on-promotion only              |
| `filters.department`    | no       |                                |
| `filters.aisle`         | no       |                                |
| `pagination.limit`      | no       | default 20, max 100            |
| `pagination.cursor`     | no       | opaque, from previous response |

### UCP response — what you must return

```json
{
  "products": [
    {
      "id": "string",
      "title": "string",
      "description": "string (optional)",
      "media": [{ "url": "string", "type": "image" }],
      "variants": [
        {
          "id": "string",
          "price": { "amount": 150, "currency": "GBP" },
          "list_price": { "amount": 200, "currency": "GBP" },
          "availability": "in_stock | out_of_stock | limited",
          "options": [{ "name": "unit", "value": "75p/100g" }]
        }
      ]
    }
  ],
  "total": 42,
  "next_cursor": "base64string (only when more pages exist)",
  "messages": [{ "type": "info", "code": "promotion", "content": "..." }]
}
```

**Critical:** `variant.id` is passed to `create_checkout line_items[].item.id`.
This is the link between catalog and checkout.

### Your Tesco GraphQL query (use this)

```graphql
query SearchProducts(
  $query: String!
  $page: Int
  $count: Int
  $offset: Int
  $brand: String
  $brands: [String]
  $dietary: String
  $dietaries: [String]
  $offers: Boolean
  $department: String
  $aisle: String
) {
  search(
    query: $query
    page: $page
    count: $count
    offset: $offset
    brand: $brand
    brands: $brands
    dietary: $dietary
    dietaries: $dietaries
    offers: $offers
    department: $department
    aisle: $aisle
  ) {
    pageInformation: info {
      totalCount: total
      pageNo: page
      count
      pageSize
      offset
    }
    productItems: products {
      id
      title
      shortDescription
      defaultImageUrl
      brandName
      isForSale
      status
      price {
        price: actual
        unitPrice
        unitOfMeasure
      }
      promotions {
        promotionId: id
        promotionType
        offerText: description
        startDate
        endDate
      }
    }
  }
}
```

Note: does NOT include `restrictions(startDateTime:...)` — that adds variable
dependencies not needed for catalog.

### Cursor encoding

Tesco uses page+offset pagination. UCP expects opaque cursor strings.

```typescript
// encode { page, offset } → base64 string passed to agent as next_cursor
// decode base64 string → { page, offset } to use in next GraphQL call
// default when no cursor: { page: 1, offset: 0 }
```

### Field mapping rules

| Tesco GQL                                 | UCP field                     | Transform                                              |
| ----------------------------------------- | ----------------------------- | ------------------------------------------------------ |
| `id`                                      | `product.id` AND `variant.id` | same value — Tesco = 1 variant per product             |
| `title`                                   | `product.title`               | direct                                                 |
| `shortDescription`                        | `product.description`         | direct                                                 |
| `defaultImageUrl`                         | `product.media[0].url`        | wrap: `{ url, type: 'image' }`                         |
| `price.actual`                            | `variant.price.amount`        | × 100 → pence (UCP uses minor units)                   |
| `price.unitPrice` + `price.unitOfMeasure` | `variant.options`             | `[{ name: 'unit', value: '75p/100g' }]`                |
| `isForSale: false`                        | `variant.availability`        | `'out_of_stock'`                                       |
| `status: 'LOW_STOCK'`                     | `variant.availability`        | `'limited'`                                            |
| otherwise                                 | `variant.availability`        | `'in_stock'`                                           |
| `promotions[].offerText`                  | `messages[]`                  | `{ type: 'info', code: 'promotion', content }`         |
| `pageInformation.totalCount`              | `total`                       | direct                                                 |
| has more pages                            | `next_cursor`                 | encodeCursor({ page: pageNo+1, offset: offset+count }) |

`in_stock_only` filter: if Tesco GraphQL doesn't support it natively,
apply client-side: `.filter(p => !args.filters?.in_stock_only || p.isForSale)`

### Implementation steps

1. Check `@ucp-js/sdk` exports — use any catalog types it has
2. Write `src/types/ucp.ts` (or import from SDK)
3. Write `src/graphql/queries/catalog.gql.ts` with query above
4. Run `npm run codegen` → generates `src/graphql/generated/catalog.types.ts`
5. Write `src/tools/ucp/catalog/tool.ts`
6. Wire into `src/tools/ucp/index.ts` and `src/tools/index.ts`
7. Write `src/index.ts`
8. `npm run dev` and test with MCP inspector

## Task 2: `lookup_catalog` (after search works)

**Spec:** https://ucp.dev/draft/specification/catalog/mcp/#lookup_catalog

Differences from search:

- Input: `ids: string[]` instead of `query`
- For each id, call search using that id as query, take first result
- Each variant gets `inputs: [{ id: requestedId, match: 'exact' }]`
- No pagination (returns exactly what was requested)
- Lives in the same `catalog/tool.ts` as a second export

## `scripts/generate-openrpc.ts`

```typescript
import { createMcpServer } from 'gql-to-mcp';
import { allTools } from '../src/tools/index.js';
import { writeFileSync } from 'fs';

async function main() {
  // no server.start() needed — listTools() reads registry directly
  const server = createMcpServer({ config: './mcp-config.yaml', tools: allTools });
  const tools = server.listTools();

  const openrpc = {
    openrpc: '1.2.1',
    info: { title: 'tesco-commerce-mcp', version: '1.0.0' },
    methods: tools.map((tool) => ({
      name: tool.name,
      description: tool.description ?? '',
      params: tool.inputSchema?.properties
        ? Object.entries(tool.inputSchema.properties).map(([k, v]) => ({
            name: k,
            required: (tool.inputSchema.required ?? []).includes(k),
            schema: v,
          }))
        : [],
      result: { name: 'result', schema: { type: 'object' } },
    })),
  };

  writeFileSync('./openrpc.json', JSON.stringify(openrpc, null, 2));
  console.log(`✅ openrpc.json — ${tools.length} tools`);
  tools.forEach((t) => console.log(`   • ${t.name}  [${t.authMode}]`));
}

main().catch(console.error);
```

## Constraints

- Never edit `graphql/generated/` files — always re-run codegen
- Never edit `openrpc.json` manually — always re-run `generate:openrpc`
- No REST endpoints — MCP only
- No separate GraphQL client — use `ctx.gql()` from SDK
- Prices in pence (minor units) in all UCP responses — `price.actual × 100`
- `variant.id` must work directly in `create_checkout line_items[].item.id`
- `authMode: 'anonymous'` for catalog, `authMode: 'jwt'` for checkout/orders
- Tool description quality matters — LLM uses it to select the right tool
